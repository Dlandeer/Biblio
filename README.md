# Biblio

**Biblio** — это сервис для управления базой данных библиотеки, поддерживающий изменение библиотечного фонда (добавление, удаление и изменение данных о книгах), авторизацию пользователей и сохраняет историю владения книгами, когда какой пользователь владел книгой.

## Решение бизнес-задачи
Проект не просто сохраняет строки (CRUD), а сохраняет историю взаимодействия пользователей с книгами:
* **Получение истории владения**: В проекте реализованны функции получения истории владения конкретной книгой, данных о том какими книгами владел конкретный пользователь и доступ к полной истории владеения, сгруппированой по пользователям.
* Функции доступа к истории владения книг реализованы в файле connections.py.

## Технологический стек
* **Фреймворк**: FastAPI
* **База данных**: PostgreSQL + SQLAlchemy 
* **Валидация**: Pydantic
* **Аутентификация**: OAuth2 с JWT-токенами, хеширование паролей (Passlib/Bcrypt)
* **Контейнеризация**: Docker & Docker Compose
* **Тестирование**: Pytest + TestClient
* **Линтинг**: Pylint (Оценка: 9.87/10, файлы pylint.txt и .pylintrc приложены)

## Архитектура БД и CRUD
Реализована полноценная реляционная база данных (3 связанные таблицы):
1. **User** (Пользователи)
2. **Book** (Книги)
3. **BookOwnship** (История владения книгами, сопоставление User и Book)

Для сущностей реализован полный набор CRUD-операций (всего **13 эндпоинтов**), включая создание, чтение (списком и детально), обновление (переименование).

## Авторизация и валидация
* **JWT / OAuth2**: Защита эндпоинтов реализуется через механизм `OAuth2PasswordBearer`. Большинство маршрутов требуют передачи Bearer-токена.
* **Pydantic**: Валидация входящих данных.

## Установка и запуск

### Запуск через Docker 
Убедитесь, что у вас установлены Docker и Docker Compose.

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Dlandeer/Biblio.git
   cd Biblio
   ```
2. Переименуйте `.env.example` в `.env`, а `portgres.env.example` в `portgres.env` и укажите ваши настройки:
   ```bash
   cp .env.example .env
   bash
   cp portgres.env.example portgres.env
   ```
3. Запустите контейнеры, если не сработало с первого раза, попробуйте второй раз в той же папке:
   ```bash
   docker-compose up --build
   ```
Приложение будет доступно по адресу: `http://localhost:8000`

## Автоматическое тестирование
Проект покрыт автоматическими тестами с использованием `pytest` и `TestClient` от FastAPI, тестирующими ключевую логику и эндпоинты.

Запуск тестов:
```bash
pytest tests/
```

## Документация API (Swagger UI)
FastAPI автоматически генерирует интерактивную документацию. После запуска сервера она доступна по адресам:
* **Swagger UI:** `http://localhost:8000/docs`
* **ReDoc:** `http://localhost:8000/redoc`
